# Extract installer from Dockerfile into bash + nu script

## Goal

Move install/settings logic out of Docker layers into a portable script. Flow: `bootstrap.sh` ensures brew + nu, then hands off to `bootstrap.nu`, which does the work. Dockerfile becomes a thin wrapper.

Future, deferred: alt-XDG-prefix mode for trying this on a host with existing configs. Don't shape PR 1 around it.

## Design rules

- **Primary target**: Docker sandbox — Docker is deprecating `docker sandbox` in favor of an `sbx` binary; cozy follows. Host install is secondary — feature parity only.
- **Feature parity** with today's Dockerfile. Claude+MCP on by default, `global-claude.md` always appended, dotfiles always deployed.
- **Idempotency** is not a goal. Re-run = clean setup.
- **No sudo prompts**. Host: assume brew installed, error otherwise.
- **No shell-rc writes** by host bootstrap.
- **Reuse what exists. Don't write parallel modules.**

## Reuse map (stuff already implemented; do not re-create)

- `toolkit/vendor.nu` — github-tarball vs `--local` rsync of vendored modules. Reads `toolkit/vendor.yml`.
- `vendor/dotfiles/toolkit.nu` — `push-to-machine` (deploys configs), `install-skills --all` (deploys `~/repos/my-claude-skills/...` and `~/repos/nushell-skills/...` into `~/.claude/skills/`).
- `sandbox-toolkit/install/topiary.nu` — topiary install + grammar build.
- `sandbox-toolkit/install/claude.nu` — claude install.

## PR 1 — single `bootstrap.nu`

One new file: `sandbox-toolkit/install/bootstrap.nu`. Everything inline; no helper modules.

Steps in order:

1. `brew install nushell fzf lazygit helix zellij broot git-delta visidata bat topiary fd` then `brew install jj git-lfs`; `brew cleanup --prune=all` after each. (Two groups for build-cache reuse, matching today's Dockerfile lines 38, 41.)
2. Write `~/.config/git/{config,ignore}` — two `printf`s' worth of content from Dockerfile lines 82–84.
3. `populate_repos --in-docker=$in_docker --local=$local`:
   - In Docker: `cp -r /tmp/vendor/* ~/repos/` and copy sandbox-toolkit (today's behavior).
   - On host, `--local`: shell out to `nu toolkit/vendor.nu --local` (rsync from sibling repos), then place results under `~/repos/`.
   - On host, default (github clone): shell out to `nu toolkit/vendor.nu` (github tarball mode), then place results under `~/repos/`.
4. `cd ~/repos/dotfiles; use toolkit.nu; toolkit push-to-machine --force --create-dirs --commit-changes` (with `--docker` when `--in-docker`).
5. `toolkit install-skills --all` (already in `vendor/dotfiles/toolkit.nu:334`).
6. Append `docker-files/global-claude.md` to `~/.claude/CLAUDE.md`.
7. `^broot --write-default-conf $XDG_CONFIG_HOME/broot; ^broot --set-install-state installed`.
8. `use topiary.nu; topiary install`.
9. `use claude.nu; claude install`, then `^claude mcp add --scope user --transport stdio nushell -- /home/linuxbrew/.linuxbrew/bin/nu --mcp`.

Wire into `install/mod.nu` next to `claude`, `topiary`, etc., so `cozy install bootstrap` works inside the sandbox.

Signature: `export def main [--in-docker --local]`. `--local` forwards to `toolkit/vendor.nu --local` for the host case (rsync from sibling repos instead of github clone). No `--no-claude`, no `--modules=…` flag — add them only when a real second caller exists.

## PR 2 — Dockerfile collapse

Replace lines 33–127 of `Dockerfile` (apart from the bits below) with:

```
RUN nu -c 'use ~/repos/cozy/sandbox-toolkit/install/bootstrap.nu; bootstrap --in-docker'
```

Stays inline in Dockerfile (Docker-isms, not install logic):

- `USER` switches.
- `ENV` for `HELIX_RUNTIME`, `HOME`, `TERM`, `COLORTERM`, `TERM_PROGRAM`, `LANG`, `XDG_*` (host already has these correct; forcing them would stomp the user's terminal/locale).
- `COPY` for vendor, sandbox-toolkit, docker-files, README.
- `pbcopy` shim, apt proxy, `/etc/gitconfig` build-time identity.
- `/etc/sandbox-persistent.sh` env exports.

(Broot init moved into `bootstrap.nu` step 7 so the host path gets it too.)

Drop `ARG MODULES_SOURCE=clone` and `ARG INSTALL_CLAUDE=true`. Add them back only when someone wants to flip them.

## PR 3 — host entry (only if actually needed)

`cozy/bootstrap.sh`, ~10 lines:

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")"
command -v brew >/dev/null || { echo "Install brew: https://brew.sh"; exit 1; }
command -v nu  >/dev/null || brew install nushell
exec nu sandbox-toolkit/install/bootstrap.nu "$@"
```

`bootstrap.sh --local` forwards `--local` to `bootstrap.nu`, which then calls `toolkit/vendor.nu --local` (rsync from sibling repos) instead of github clone. No `--in-docker` autodetect. Docker keeps calling `nu -c` directly (PR 2). Host calls `bootstrap.sh`. They share `bootstrap.nu` — that's the whole point.
