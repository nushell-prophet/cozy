# Extract installer from Dockerfile into bash + nu script

## Goal

Move install and settings logic out of Docker layers into a portable script that runs on macOS/Ubuntu via plain `bash`. Flow: `bootstrap.sh` ensures Homebrew + nushell, then hands off to `bootstrap.nu` which installs everything else and applies settings. The Dockerfile becomes a thin wrapper that calls the same script.

Future (separate work): support installing into an alternative XDG prefix so the script can run on a host that already has its own configs.

## Design rules

- **Primary target**: Docker sandbox (the upcoming `sbx` binary, deprecating `docker sandbox`). Host install on macOS/Ubuntu is a secondary goal — feature parity, not first-class polish.
- **Feature parity** with today's Dockerfile by default. Claude Code + MCP install ON, `global-claude.md` always appended, dotfiles always deployed.
- **Idempotency** is not a goal. Re-run = clean setup. Don't add skip-guards (existing ones in `topiary.nu`/`nushell.nu` stay since they don't block re-run; just don't add more).
- **No sudo prompts** anywhere. macOS host: assume brew already installed (else error with a one-line install hint). Docker: brew install via `NONINTERACTIVE=1 curl|bash` as today.
- **No shell-rc writes** by host bootstrap. `XDG_*`, `HELIX_RUNTIME`, `TERM`, `LANG`, `HOME` stay Docker-only ENV in the Dockerfile.

## Layout

```
bootstrap.sh                       # bash entry — ensures brew + nu, hands off
sandbox-toolkit/
  install/
    bootstrap.nu                   # NEW: orchestrator (brew tools, modules, dotfiles, skills, ...)
    brew-tools.nu                  # NEW: brew install <list>
    modules.nu                     # NEW: clone-from-github OR rsync-from-local for ~/repos/*
    dotfiles.nu                    # NEW: invoke vendored dotfiles toolkit push-to-machine
    skills.nu                      # NEW: deploy ~/.claude/skills/
    git-xdg.nu                     # NEW: write ~/.config/git/{config,ignore}
    claude.nu                      # exists — also registers nushell MCP
    topiary.nu                     # exists
    nushell.nu, polars.nu, ...     # exist; opt-in subcommands, not part of default 'all'
```

`bootstrap.sh` stays small (~30 lines). All real work lives in nu modules.

## Step ordering inside `bootstrap.nu`

1. `brew-tools install`
2. `git-xdg apply`
3. `modules fetch` — clone (or rsync from `--local`) into `~/repos/`
4. `dotfiles deploy` — runs `toolkit push-to-machine --force --create-dirs --commit-changes` (adds `--docker` when `--in-docker`)
5. `skills deploy`
6. Append `global-claude.md` to `~/.claude/CLAUDE.md`
7. `topiary install`
8. `claude install` + nushell MCP register (gated by `--no-claude`)

Each sub-step prints a one-line summary.

## PR sequence

### PR 1 — extract installer modules + `bootstrap.nu` orchestrator

Files to create under `sandbox-toolkit/install/`:

- `brew-tools.nu` — `export def install []` runs `brew install <list>` from a `const tools = [...]` list, splitting into the same two groups as today (Dockerfile lines 38, 41) so build cache stays hot. `brew cleanup --prune=all` after each group.
- `modules.nu` — `export def fetch [--source: string = "github"]` where source is `github` or `local`. Mirrors `toolkit/vendor.nu --local` rsync logic for `local`; clones the URL list for `github`. Always also copies `sandbox-toolkit/` into `~/repos/cozy/sandbox-toolkit/`. Reads vendored module list from `toolkit/vendor.yml` (single source of truth) when running in `github` mode.
- `dotfiles.nu` — `export def deploy [--in-docker]` runs `cd ~/repos/dotfiles; use toolkit.nu; toolkit push-to-machine --force --create-dirs --commit-changes`, adding `--docker` when `--in-docker`.
- `skills.nu` — copies `~/repos/my-claude-skills/plugins/my-skills/skills/*` and `~/repos/nushell-skills/plugins/*/skills/*` into `~/.claude/skills/`.
- `git-xdg.nu` — writes `~/.config/git/{config,ignore}` with the same content as Dockerfile lines 82–84.
- `bootstrap.nu` — `export def main [--in-docker --modules: string = "github" --no-claude]` runs the 8 ordered steps. Default is feature parity (claude on, github clone).

Wire into `install/mod.nu` so `cozy install bootstrap` works inside the sandbox.

Dockerfile change in PR 1: replace inline `RUN brew install ...`, `RUN cp -r /tmp/vendor/...`, `RUN cd ~/repos/dotfiles && nu -c ...`, the skills `cp -r`, the `git config` block, and the topiary install line with a single `RUN nu -c 'use ~/repos/cozy/sandbox-toolkit/install/bootstrap.nu; bootstrap --in-docker --modules=vendor'`. Modules still come from `/tmp/vendor` because `modules.nu` `vendor` mode reads it.

Build still works; Dockerfile is shorter; no behavior change.

### PR 2 — `bootstrap.sh` bash entry

`cozy/bootstrap.sh` (~30 lines):

```bash
#!/usr/bin/env bash
set -euo pipefail

cd "$(dirname "$0")"

in_docker=0
[[ -f /etc/sandbox-persistent.sh || "${COZY_IN_DOCKER:-}" == "1" ]] && in_docker=1

if ! command -v brew >/dev/null 2>&1; then
  if [[ $in_docker -eq 1 ]]; then
    NONINTERACTIVE=1 /bin/bash -c \
      "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  else
    echo "ERROR: Homebrew not found. Install from https://brew.sh first." >&2
    exit 1
  fi
fi

[[ -d /home/linuxbrew/.linuxbrew/bin ]] && export PATH="/home/linuxbrew/.linuxbrew/bin:$PATH"
[[ -d /opt/homebrew/bin ]] && export PATH="/opt/homebrew/bin:$PATH"

brew list nushell >/dev/null 2>&1 || brew install nushell

exec nu sandbox-toolkit/install/bootstrap.nu --in-docker="$in_docker" "$@"
```

Verify on:
- Clean Ubuntu container (Linuxbrew path).
- macOS (assumes brew present).
- Docker build (PR 3 will switch the Dockerfile to use it).

### PR 3 — Dockerfile uses `bootstrap.sh`

Dockerfile shrinks to:

1. Base image + apt deps + `pbcopy` shim + apt proxy + `/etc/gitconfig` build-time fallback (root).
2. `USER agent` + ENV vars (`HELIX_RUNTIME`, `HOME`, `TERM`, `COLORTERM`, `TERM_PROGRAM`, `LANG`, `XDG_*`).
3. `COPY` repo into image (vendor + sandbox-toolkit + bootstrap.sh + docker-files).
4. `RUN bash bootstrap.sh --modules=vendor`.
5. `RUN echo … >> /etc/sandbox-persistent.sh` (env exports).
6. `COPY README.md` to workspace.

Build args (`MODULES_SOURCE`, `INSTALL_CLAUDE`) become `bootstrap.sh` flags forwarded from `ARG` lines.

## Implementation notes

- `bootstrap.sh` arg style: long flags (`--in-docker --modules=vendor --no-claude`) — match the nu side.
- `modules.nu` source-of-truth: read `toolkit/vendor.yml` so the URL list lives in one place. Dockerfile clone block today hardcodes URLs duplicated elsewhere — fold them into `vendor.yml` if needed (extra `github` field per entry already exists), or keep a separate URL field.
- `--docker` flag passed to dotfiles' `push-to-machine` (Dockerfile line 110): keep its behavior identical; the only difference is whether it's set.
